Android Studioでいつの間にかJDKが消えた件と対応

  • tech 
  • android 

どのタイミングからなのかわかりませんが（おそらく2020.3.1以降）、Android Studioに埋め込まれていたJDKのパスが変わってました。

AS付属のJDKのパスは、Preferencesの「Build, Execution, Development」＞「Build Tools」＞「Gradle」にあるGradle JDKで確認できます。

どうやらAS内からLibraryに移動したようですね。 ですので、 zshやbashの設定ファイルで設定した、JAVA_HOMEやPATHのパスを修正します。

export PATH=$PATH:/Library/Java/JavaVirtualMachines/jdk-16.0.2.jdk/Contents/Home/bin
export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-16.0.2.jdk/Contents/Home/

設定ファイルを読み直し、 java --version でちゃんとjavaの読み込みが確認できたので本来はこれで対応は終わります。

BUILD FAILED in 16s

stackoverflowに同じ悩みを抱えた投稿がありましたが、どうもJDK 16だとビルドに失敗してしまうようです。

How to fix the ‘‘module java.base does not “opens java.io” to unnamed module '’ error in Android Studio? - Stack Overflow

非常に腑に落ちないですが、JDK 15をインストールすることにします。 JDKのインストールですが、先程のASの設定からGradle JDKのコンボボックスをクリックすると「Download JDK」という項目があり、ここから好きなバージョンを選んでインストールできます。

ということで、あらためてJDK 15をインストールし、JAVA_HOMEとPATHに反映させたところ無事ビルドが通るようになりました。 めでたし。
